I. The Problem¶

Build and present a model predicting the Overall Amazon reviews' rating

Implementing a solution to this problem involves:

  1. Downloading the Amazon reviews data
  2. Repackaging JSON into a workable format
  3. Preparing and preprocessing the review data (features) and the overall ratings (labels)
  4. Training a model on the prepared features to predict the labels (ratings)
  5. Testing the model performance on an "unseen" subset of data

II. A one-day Proof-of-Concept and its implementation¶

Overview¶

Due to time and computing resource limitations, we simplify the problem slightly:

  1. we only predict three rating classes:
    • Negative [1/5]
    • Slightly negative to neutral [2/5, 3/5]
    • Positive [4/5, 5/5]
  2. we limit ourselves to using the full-text reviews
  3. we implement only simple methods to prevent overfitting (more advanced method discussed below)
  4. we do not attempt optimization of hyperparameters
  5. we do not implement advanced performance assessments such as K-fold cross-validation. Instead, we test our model on a third set of independent data (test set).
Proprocessing steps¶
  1. The Data is read and repackaged into pandas DataFrame
  2. First step of data clean-up
    • remove duplicate entries
    • remove rows with no reviews or overall rating
  3. Extract helpfulness and create a "helpfulness score."
  4. Preprocess text reviews:
    • remove contractions
    • make lower-case
    • remove punctuations
    • do not remove stop words (removing creates issues with negative reviews)
    • word lemmatization, i.e., modelled --> model, modelling --> model
Leveraging the state-of-the art: BERT¶

BERT is a pre-trained language model for NLP created by Google AI language. The key innovation is applying a bidirectional encoder, i.e., the words are interpreted in context (left and right).

Fine tuning of BERT¶
  • We reserve data for testing by sampling 10000 reviews without replacement
  • We sample an addional 10000 review for training and split it into:
    • a training set (80%)
    • a validation set (20%)
  • We extract features from preprocessed text using BERT tokenizor
  • We train the model to predict three rating classes:
    • 'negative' [1/5]
    • 'neutral' [2/5, 3/5]
    • 'positive' [4/5, 5/5]

Hyper parameters¶

Due to memory limitation on personal GPU:

- we are not able to use large batch sizes
- we cannot run for many epochs
- we cannot perform an optimization of hyperparameters
  • we use suggested initial parameters
    • learning rate = 5e-5
    • shuffle = 100
  • We use a batch size = 4.
  • We run for 5 epochs
Handling overfitting¶

To prevent the model from overfitting, we use a dropout layer for regularization (30% probability) and a validation-stop: the training stops if the validation loss has not improved over two epochs.

There are several other ways to prevent overfitting:

  • L1 and L2 regularization
  • train an ensemble of models with varying parameters and data samples
  • tuning the parameters of the dropout layer

My favorite is to use dropout, which is usually a low-effort yet powerful way to prevent overfitting. Ensemble modeling is interesting because the spread of predictions can indicate uncertainty but requires a lot of computing resources.

III. Improving the solution¶

  1. Leveraging review helpfulness as well as both long and short reviews
    • Train a model to predict unhelpfulness and in order to discard unhelpful reviews.
    • Use helpfulness, short and long reviews prediction as inputs to a meta-classifier model.

  2. Increase computing resources:

    • Optimization of model hyperparameters
      • batch size
      • learning rate
    • Train on larger set of data and with increased maximum length
    • Train for more epochs

  3. Explore and tune several regularization methods

    • Tune dropout layer parameters
    • Test and tune L1 and L2 regularization
  4. Multi-model prediction:
    • Train an ensemble of models with varying hyperparameters, samples, pre-processing.
    • Stack: use a meta-model to predict from an ensemble of sub-model predictions (varying hyperparameters, data sample, pre-processing)most likely label as the prediction.
    • Use clustering algorithm to pre-process data or extract new features:
      • E.g., use K-means to cluster users into groups
      • Use clusters as features in the training chain or use the clusters to train seperate models
